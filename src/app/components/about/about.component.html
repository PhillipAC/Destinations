Destinations

About

Destinations is an application designed for setting up procedurally generated routes for use in live action games. A simple example would be creating a route for people who are playing tag to go between without being caught by the person who trying to tag them.

Editor

The Editor allows someone to modify the configuration for generating the game. The area can be split into 4 general areas

Game Info:
The Game Info section allows you to change nomenclature that is used for the game.

Areas:
Areas are large sections of the game that contain at least one location

Locations:
Locations are places that be selected as a destination Adjacent Areas:
Unless there are no other options, an area selected will not be adjacent to the previous destination's area

Example Configurations

While Destinations is designed to allow anyone to create their own custom configuration for a game, a set of pre-defined configurations can be loaded

Stalker:
A surivial horror style game of hide and seak played a house with all the lights off. The hiders start in the first location. Then the seaker (Stalker) enters the game area. While the Stalker moves around the building the hiders sneak around and attempt to move to their defined locations. They win if they can "escape" by getting to the last area.

Tag:
A game of tag; however, the people who are running now have destinations to try and get to. All runners start at the first location and then attempt to get to each destination. If they can get to each destination they win the round. If the tagger tags them then they lose the round.

City Walk: Less of a game and more of a exercise. This configuration shows how to use destinations to create a route that one could go on for a nice stroll around a town. Start at the first location and move between the different destinations until you get to the final location. No lossing, no winning, just a nice time outside.
